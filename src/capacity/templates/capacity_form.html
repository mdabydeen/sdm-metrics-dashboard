📊 Sprint Capacity Input

Record engineer availability for sprints. This data powers capacity utilization and ETA predictions.

Enter Capacity

Quick Reference

Available Days: Working days the engineer has available (subtract PTO, on-call, and planned meetings).

Total Days: Total working days in the sprint (usually 10).

Capacity Points: Expected story points the engineer can complete (optional, helps validate velocity).

Notes: Anything affecting capacity — on-call rotation, mentoring, training, etc.

Import from CSV

Upload a CSV file with the following columns:
sprint_id,engineer_id,available_days,total_days,capacity_points,notes
